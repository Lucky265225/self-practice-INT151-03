Synchronous
- คือรูปแบบการทำงานที่ ต้องรอให้คำสั่งปัจจุบันเสร็จก่อน ถึงจะไปทำคำสั่งถัดไป
- ทำงาน ทีละงาน เป็นลำดับ
- ถ้างานหนึ่งช้า -> งานต่อไปต้องรอ
- ปกติในโปรแกรมทั่วไปหรือโค้ดแบบพื้นฐาน


Asynchronous
- คือรูปแบบการทำงานที่ ไม่ต้องรอให้คำสั่งก่อนหน้าจบ แต่ปล่อยให้ระบบทำงานเบื้องหลัง และไปทำงานถัดไปได้เลย
- ทำงาน พร้อมกันหลายงานได้
- เพิ่มความเร็ว เมื่อมีงานที่ต้องรอ เช่น network I/O, database, file
- ใช้ในเว็บเซิร์ฟเวอร์, แอปที่ต้องตอบสนองเร็ว


Promise
- ใช้ในการจัดการ Asynchronous
- เป็นการบอกว่าจะได้ผลลัพธ์ในอนาคต
- มี 3 สถานะหลักๆ pending(รอผล), fulfilled(สำเร็จ), rejected(ล้มเหลว)


ความรู้ที่ได้รับ
- เข้าใจวิธีการเขียนโค้ดมากขึ้น
- สามารถนำไปใช้เวลาทำงานกับ server เวลาต้องรอข้อมูลมา
- ช่วยให้โค้ดสามารถทำงานได้โดยลำดับการทำงานยังถูกต้อง
