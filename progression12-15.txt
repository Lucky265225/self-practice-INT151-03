Synchronous
- ทำงานทีละงาน เป็นลำดับ
- ต้องรอคำสั่งปัจจุบันเสร็จก่อนถึงทำงานถัดไป

Asynchronous
- ไม่ต้องรอคำสั่งก่อนหน้าเสร็จ
- ทำงานหลายงานพร้อมกันได้

Promise
- ใช้จัดการงาน Asynchronous
- บอกว่า “จะได้ผลลัพธ์ในอนาคต”

Fetch API
- ใช้ ส่งคำขอไปยัง server และรับ response

Date Object
- รู้การสร้าง
- เข้าใจรูปแบบ
- รู้วิธีการเปรียบเทียบ

Dialog
- รู้การทำงาน
- ประเภทและความแตกต่างของ dialog 


สรุปประโยชน์ที่ได้จากการเรียนรู้
- เข้าใจ flow ของโค้ดทั้ง synchronous และ asynchronous
- จัดการการทำงานกับ server และ fetch ข้อมูล CRUD ได้
- สร้างและแสดงผลวันเวลาอย่างเหมาะสม
- เลือกใช้ dialog ในเว็บตามความเหมาะสม
- สามารถนำความรู้ไปใช้ในโปรเจค Integrated และงานจริง
